// 元素节点 // 节点是网络术语，表示网络中的一个连接点。一个网络节点就是由一些节点构成的集合 // 文档是节点构成的集合，只不过此时的节点文档树上的树枝和枝叶 // 节点分为三种： 元素节点、文本节点、和属性节点 // 元素节点 // DOM的原子就是元素节点，标签的名字就是元素的名字 // 如果web上的文档比做一座大厦的话，元素就是减灾这座大厦的砖块，这些元素的布在文档中的布局形成了文档的结构 // 标签的名字就是元素的名字，并且元素可以包含其他元素 // 文本节点 // 如果一个文档完全由一些空白的元素构成，他将有一个结构，但是不会包含任何内容。 // 在xhtml中，文本节点总是被包含在元素节点的内部。但并非所有的元素节点都包含有文本节点。 // 属性节点 // 属性节点用来对元素作出更具体的描述 // css // 继承是css技术中的一项强大功能。类似于DOM，CSS也把文档的内容视为一颗节点树。节点上的各个元素将继承父元素的样式属性。 // 获取元素 // getElementById 返回给定Id属性值的元素节点对应的对象 // JS 区分大小写 // getElementByTagName 返回一个对象数组。每个对象分别对应着文档中有着给定标签的一个元素，他的参数是标签的名字 e.g. li\div\span // 该方法还允许将一个通配符 * 作为他的参数。那么这意味着，他将返回文档中的所有元素 // getElementByClassName 这个方法能通过Class属性中的类名来访问元素，返回具有相同类名的数组。 // 获取和设置属性 // getAttribute 是一个函数，只有一个参数，查询的属性名 // 这个方法不属于document对象，所以不能通过document来调用。 // 他只能通过元素节点来调用 // setAttribute 允许我们对元素节点进行修改，只能由元素节点来调用。 // setAttribute 作出的修改不会反映在文档本身的源码里，这种表里不一的现象源自于Dom的工作模式， // 先加载文档的静态内容，再动态刷新，动态刷新不影响文档的静态内容。 // 所以，对页面内容进行刷新却不需要在浏览器中刷新页面 // // window.open(url, name, feature) // url 新打开的窗口的地址 // name 新窗口的名字 // 以逗号分割开的字符串 // window.open(windURL,"poopup","windth=200,height=200") // JS伪协议 -- “javascript：” // 他让我们通过一个连接来调用js函数 example 其中popup是一个js函数 // 一般来说，根据HTTP规范，浏览器每次从一个域名中最多只能同时下载两个文件，而在下载期间，浏览器不会下载其他任何文件，即使是来自不同域名的文件也不会下载，所有其他资源都要等脚本加载完毕之后才能下载。
